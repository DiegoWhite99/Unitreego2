1. Descripcion General del Proyecto

Daiver Control CUN es una aplicacion web completa para el control remoto del robot cuadrupedo Unitree Go2 Air a traves del protocolo WebRTC. El sistema permite a los usuarios interactuar con el robot desde cualquier navegador web moderno, proporcionando control de movimiento en tiempo real, ejecucion de poses y trucos, y rutinas autonomas preprogramadas.

El proyecto ofrece dos interfaces diferenciadas:

Interfaz URL Usuarios Objetivo Caracteristicas Principales
Dashboard / Operadores tecnicos Control preciso de movimiento, telemetria, logs en tiempo real
Landing (Daiver) /landing Publico general Interfaz amigable, sintesis de voz, botones de acciones con feedback

2. Arquitectura del Sistema

El sistema sigue una arquitectura cliente-servidor con comunicacion bidireccional en tiempo real:

NAVEGADOR WEB (Chrome / Edge / Firefox)
Dashboard (index.html)  |  Landing Page (landing.html)
↓ HTTP + Socket.IO ↓
SERVIDOR FLASK (app.py — Puerto 5000)
API REST + Socket.IO + Archivos Estaticos
↓ WebRTC (aiortc) ↓
UNITREE Go2 AIR (IP: 192.168.1.36)
Sport Mode Commands via WebRTC PubSub

2.1 Flujo de Comunicacion

  1. El usuario interactua con la interfaz web (botones, teclado, D-Pad).
  2. Las acciones se envian al servidor Flask mediante HTTP REST (acciones puntuales) o Socket.IO (movimiento continuo).
  3. El servidor Flask traduce las peticiones a comandos SPORT_CMD del robot.
  4. Los comandos se envian al Go2 Air via WebRTC usando la libreria unitree_webrtc_connect.
  5. El robot ejecuta el comando y el servidor actualiza el estado global.
  6. El estado actualizado se propaga a todos los clientes conectados via Socket.IO.

2.2 Modelo de Concurrencia

El servidor utiliza un hilo dedicado con un event loop asyncio para las operaciones WebRTC, separado del hilo principal de Flask. Esto permite que las rutinas autonomas (que pueden durar varios minutos) se ejecuten sin bloquear las peticiones HTTP del servidor. 

# Modelo de hilos
Hilo Principal  → Flask + Socket.IO (peticiones HTTP, eventos WebSocket)
Hilo Asyncio   → Event loop dedicado (WebRTC, rutinas autonomas, movimiento)

4.2 Estado Global del Robot

El servidor mantiene un diccionario global robot_state que sincroniza el estado entre todos los clientes:

robot_state = {
    "connected": False,          # Estado de conexion WebRTC
    "ip": "192.168.1.36",        # IP del robot
    "battery": 0,                # Nivel de bateria (%)
    "speed": 0.64,               # Velocidad actual (m/s)
    "mode": "idle",              # Modo actual
    "current_routine": None      # Rutina en ejecucion
}

4.3 Endpoints de la API REST

Metodo Endpoint Descripcion
POST /api/connect Establece conexion WebRTC con el robot
POST /api/disconnect Desconecta del robot de forma segura
GET /api/status Retorna estado actual del robot
POST /api/command Envia comando SPORT_CMD directo
POST /api/move Movimiento continuo (velocidades x, y, z)
POST /api/stop Detiene todo movimiento
POST /api/emergency Parada de emergencia (Damp)
POST /api/action Ejecuta accion predefinida (saludar, bailar, etc.)
POST /api/routine Inicia rutina autonoma
POST /api/routine/stop Detiene rutina en ejecucion
POST /api/config/ip Actualiza IP del robot
GET /api/config Retorna configuracion de comandos disponibles

4.4 Funcion de Movimiento

El robot Unitree Go2 requiere comandos de movimiento continuos (cada 200ms) para mantener el desplazamiento. Un solo comando no genera movimiento sostenido. 

async def robot_mover(x, y, z, duracion):
    """Envia comandos de movimiento cada 0.2s durante la duracion indicada."""
    pasos = int(duracion / 0.2)
    for _ in range(pasos):
        if stop_routine_flag:
            break
        robot_pub_sub.publish("rt/api/sport/request", {
            "header": {"identity": {"api_id": 1008}},
            "parameter": json.dumps({"x": x, "y": y, "z": z})
        })
        await asyncio.sleep(0.2)

8. Rutinas Autonomas

8.1 Patrullaje (Patrol)

Recorrido en zigzag de 8 pasos que cubre un area rectangular:

  1. Avanzar 5.5 metros hacia adelante
  2. Girar 180 grados
  3. Avanzar 5 metros
  4. Girar 180 grados
  5. Avanzar 3.75 metros
  6. Girar 180 grados
  7. Avanzar 4.8 metros (retorno)
  8. Girar 180 grados (posicion original)

8.2 Salto (Jump)

Secuencia de 11 pasos combinando avance y saltos:

  1. BalanceStand (estabilizar)
  2. Avanzar 2 metros
  3. FrontJump
  4. BalanceStand
  5. Avanzar 2 metros
  6. FrontJump
  7. BalanceStand
  8. Girar 180 grados
  9. Avanzar 4 metros (retorno)
  10. FrontJump final
  11. BalanceStand

8.3 Exploracion (Explore)

Recorrido perimetral en forma de cuadrado:

  1. Avanzar 3 metros → Girar 90 grados derecha
  2. Avanzar 3 metros → Girar 90 grados derecha
  3. Avanzar 3 metros → Girar 90 grados derecha
  4. Avanzar 3 metros → Girar 90 grados derecha (retorno)
Importante: Todas las rutinas verifican la bandera stop_routine_flag entre cada paso. Si se presiona el boton de parada de emergencia, la rutina se interrumpe inmediatamente de forma segura.

9.2 Instalacion

1 Clonar el repositorio

$ git clone <url-del-repositorio>
$ cd unitreeWebRTC

2 Crear un entorno virtual (recomendado)

$ python -m venv venv

# Activar en Windows:
$ venv\Scripts\activate

# Activar en Linux/Mac:
$ source venv/bin/activate

3 Instalar dependencias de Python

$ pip install -r requirements.txt

4 Instalar el SDK de Unitree WebRTC

# Instalar la libreria de comunicacion WebRTC de Unitree
$ pip install unitree_webrtc_connect

# Dependencias adicionales que puede requerir:
$ pip install aiortc aioice numpy

5 Configurar la IP del robot

Editar el archivo config/config.py con la IP del robot Go2 Air:

# config/config.py
ROBOT_IP = "192.168.1.36"   # Cambiar si la IP es diferente
Nota: La IP tambien puede cambiarse desde la interfaz web en el Dashboard, usando el formulario de conexion o el endpoint POST /api/config/ip.

6 Conectar a la red WiFi del robot

  • Encender el Unitree Go2 Air.
  • Esperar a que el robot emita su red WiFi.
  • Conectar el computador a la red WiFi del Go2 Air.
  • Verificar conectividad: ping 192.168.1.36

9 Conectar al robot desde la interfaz

  1. En el Dashboard, verificar que la IP mostrada es correcta.
  2. Hacer clic en el boton "Conectar".
  3. Esperar a que el indicador de estado cambie a CONECTADO.
  4. El robot ejecutara automaticamente un BalanceStand al conectarse.
Listo! El robot esta conectado y puedes empezar a controlarlo desde la interfaz web. Usa el D-Pad, teclado (WASD), o los botones de acciones y rutinas.

10. Guia de Uso de la Aplicacion

10.1 Controlar Movimiento (Dashboard)

  1. Asegurar que el robot esta conectado (indicador verde).
  2. Con D-Pad: Hacer clic o mantener presionados los botones direccionales.
  3. Con Teclado: Usar las teclas WASD para moverse, Q/E para girar, Espacio para detener.
  4. Velocidad: Ajustar el slider de velocidad (10% - 100%).
  5. Parar: Hacer clic en STOP o presionar Espacio.

10.2 Ejecutar Acciones

  1. Navegar a la seccion de "Acciones" o "Trucos".
  2. Hacer clic en el boton deseado (Saludar, Bailar, Saltar, etc.).
  3. El robot ejecutara la accion y el log mostrara el resultado.

10.3 Iniciar una Rutina

  1. Asegurar que el area esta despejada (el robot se movera de forma autonoma).
  2. Seleccionar la rutina deseada (Patrullaje, Salto o Exploracion).
  3. Monitorear el progreso en los logs en tiempo real.
  4. Para detener la rutina: presionar "Detener Rutina" o el boton de emergencia.

10.4 Usar el Landing Page (Daiver)

  1. Acceder a http://localhost:5000/landing.
  2. El sistema de voz se inicializa automaticamente.
  3. Probar las frases rapidas para escuchar a Daiver hablar.
  4. Ejecutar acciones con los botones (cada accion incluye voz).
  5. Ajustar la velocidad de la voz con el control deslizante.

10.5 Parada de Emergencia

IMPORTANTE: En cualquier momento, presionar el boton rojo de EMERGENCIA detendra inmediatamente el robot enviando el comando Damp. Este comando desactiva los motores de forma suave. La parada de emergencia funciona incluso durante la ejecucion de rutinas.

13. Soluciones a Problemas Comunes

ProblemaCausa ProbableSolucion
No se puede conectar al robot No esta en la red WiFi del Go2 Conectarse a la WiFi del robot y verificar con ping 192.168.1.36
El robot no se mueve No esta en BalanceStand Enviar comando BalanceStand antes de mover
La voz no funciona Navegador sin soporte Speech API Usar Chrome o Edge. Verificar permisos de audio
Socket.IO no conecta Puerto 5000 bloqueado Verificar firewall y que el servidor esta corriendo
Error al instalar dependencias Versiones incompatibles Usar Python 3.8+ y crear entorno virtual limpio
El robot cae o pierde equilibrio Superficie irregular Operar en superficie plana. Usar RecoveryStand para levantarse
La rutina no se detiene Boton incorrecto Usar el boton de Emergencia (rojo) que envia Damp
